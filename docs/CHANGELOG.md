## 2026-06-12  Slice 3.1: picker ranking + skipped days

Adds two engine rule areas as new `docs/engine.md` sections (the spec-code parity section renumbers to stay last, all cross-references updated, the parity module map extended). **Picker ranking** (`engine/src/pickerRanking.ts`) is a deterministic swap/add ranking over the broad, non-restrictive pool (Active + in-season + meal-time, per Principle 4), distinct from §4 generation priority: a head of "fits this day" dishes (meal-time match, not already on the day) ordered by a lexicographic tuple `(recencyTier, proteinBandDistance, id)`, then a tail of same-day repeats ordered the same way. `recencyTier` is a coarse longest-unused bucket (every never-cooked dish shares the best tier; cooked dishes tier by last-cooked weekStart) so the protein-band term genuinely orders within a tier without ever overriding recency; `proteinBandDistance` (swaps only) pulls candidates in the outgoing dish's per-person protein band (derived via `nutrition.ts`, 5 g buckets) toward the top; `id` is the final total tie-break. No RNG. `getSlotAlternatives` in `app/convex/swap.ts` switches from §4 `rankCandidates` to `rankPickerAlternatives` (ranking only; the broad-pool eligibility filter is untouched) so spec and shipped behavior match from day one. **Skipped days** makes grocery aggregation (`engine/src/groceryList.ts`) and a new pure finalize history-rows derivation (`engine/src/historyRows.ts`, `deriveHistoryRows`) skip-aware via an optional `skippedDays` input (default none, additive, every existing caller unchanged): a skipped day keeps its generated dishes in the data but contributes zero grocery rows and zero history rows (so recency never sees uncooked dishes). Generation itself is untouched; the Convex `skippedDays` schema (4.1) and the PWA "Skipped" rendering (5.x) wire these later.
